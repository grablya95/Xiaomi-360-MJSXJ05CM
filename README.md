Стабильная сборка для камеры Xiaomi Mijia MJSXJ05CM / IPC019 на базе прошивки `3.5.1_0052`.

Эта версия собрана после серии практических тестов на реальной камере. Основная цель сборки — нормальный RTSP/ONVIF-поток со звуком, рабочее двухстороннее аудио в Onvifer, веб-интерфейс, FTP-обновления и удобные настройки без постоянного вытаскивания microSD.

Сервисное исправление v1.2.2: в полный архив добавлены совместимые имена `rtspserver`, `rtspserver-v037-uclibc` и `rtspserver-v041-uclibc`, чтобы RTSP корректно стартовал после чистой установки. Installer также стал fail-safe: при критической ошибке он убирает `.force_install` и перезагружает камеру, чтобы не получить вечный цикл “web на несколько секунд → всё пропало”.

## Требуемая версия прошивки камеры

Для работы этого hack нужна штатная прошивка камеры:

```text
IPC019 / MJSXJ05CM — 3.5.1_0052
```

В полный архив v1.2.2 добавлен recovery-файл этой прошивки:

```text
firmware/IPC019_3.5.1_0052/tf_recovery.bin
```

Если на камере уже стоит `3.5.1_0052`, перепрошивать её заново не нужно — можно сразу ставить hack.

Если стоит другая версия, сначала сделайте откат на `3.5.1_0052`, а уже потом устанавливайте hack.

Важно: откат прошивки может сбросить настройки камеры. Не выключайте питание во время прошивки.

## Возможности

- RTSP-поток: `rtsp://CAMERA_IP:8554/mainstream`
- ONVIF Device Service: `http://CAMERA_IP:5000/onvif/device_service`
- Видео с камеры через RTSP/ONVIF.
- Аудио с микрофона камеры в RTSP/ONVIF.
- RTSP Backchannel для двухсторонней связи, проверено в Onvifer.
- Веб-интерфейс на `http://CAMERA_IP/` с авторизацией.
- Веб-статус: сервисы, CPU, RAM, load average.
- Файловый менеджер для microSD.
- Обновление через веб-интерфейс или FTP.
- FTP на порту `2121`, корень `/mnt/sdcard`.
- Выбор проверенных режимов видео: 20 или 25 FPS.
- LED-переключатель управляет синим и жёлтым индикаторами сразу и сохраняет состояние после перезагрузки.
- WDR действительно переключается в ISP без перезагрузки камеры.
- Установщик останавливает незавершённый старый `init.sh`, поэтому полная переустановка не зависает при размонтировании `/mnt/data`.
- Настройки из веб-интерфейса:
  - RTSP/ONVIF;
  - двухстороннее аудио;
  - FTP;
  - PTZ;
  - таймзона;
  - LED;
  - full color;
  - night vision;
  - WDR;
  - watermark;
  - video flip;
  - задержки старта аудио/backchannel.
- PTZ по умолчанию выключен. Если PTZ выключен, камера не делает стартовую проверку мотора после перезагрузки.
- Если PTZ включить в веб-интерфейсе, включается штатный режим мотора и ONVIF PTZ.

## Важные порты

| Сервис | Порт | Адрес |
| --- | ---: | --- |
| Web | 80 | `http://CAMERA_IP/` |
| ONVIF | 5000 | `http://CAMERA_IP:5000/onvif/device_service` |
| RTSP | 8554 | `rtsp://CAMERA_IP:8554/mainstream` |
| FTP | 2121 | `ftp://CAMERA_IP:2121/` |

## Установка с microSD

Нужна камера с прошивкой `3.5.1_0052`.

1. Распакуйте полный архив `MJSXJ05CM-v1.2.2-stable-full.zip` в корень microSD.
2. Откройте `hacks/installer/config.sh`.
3. Заполните свои значения:

   ```sh
   WIFI_SSID="..."
   WIFI_PASSWORD="..."
   ONVIF_USERNAME="admin"
   ONVIF_PASSWORD="..."
   TIMEZONE="Europe/Kyiv"
   ```

4. Выключите камеру.
5. Вставьте microSD.
6. Включите камеру и подождите 3–4 минуты.
7. Откройте веб-интерфейс: `http://CAMERA_IP/`.

Первый запуск не мгновенный: RTSP/ONVIF стартуют с задержкой, чтобы звук и backchannel поднимались в правильном порядке.

## Откат штатной прошивки на 3.5.1_0052

Этот шаг нужен только если камера не на версии `3.5.1_0052`.

1. Возьмите файл из архива:

   ```text
   firmware/IPC019_3.5.1_0052/tf_recovery.bin
   ```

2. Скопируйте `tf_recovery.bin` в корень microSD.
3. На карте в этот момент лучше оставить только этот файл, без папок `hacks` и `manu_test`.
4. Полностью выключите камеру.
5. Вставьте microSD.
6. Включите камеру.
7. Ждите несколько минут. Во время прошивки индикатор обычно горит жёлтым.
8. Когда камера перезапустится, начнёт крутиться и попросит QR-код — откат завершён.
9. Выключите камеру, очистите microSD от `tf_recovery.bin`.
10. После этого ставьте hack по инструкции выше.

Если откат через microSD не запускается, возможно, ваша текущая прошивка закрыла этот способ восстановления. В таком случае на GitHub у оригинального проекта писали, что может понадобиться аппаратная прошивка через программатор.

## Обновление уже установленной камеры

Если эта сборка или похожий hack уже установлен, можно использовать соответствующий update-архив без `config.sh`:

- через веб-интерфейс: загрузите update-архив;
- или через FTP:

  ```powershell
  curl.exe --user root: -T "MJSXJ05CM-update.zip" "ftp://CAMERA_IP:2121/update.zip"
  ```

После загрузки перезагрузите камеру.

Update-архив не содержит `hacks/installer/config.sh`, чтобы не перезаписать ваш текущий Wi-Fi, пароль, таймзону и другие настройки.

## Настройка Onvifer

Добавляйте камеру как ONVIF-устройство:

- IP: `CAMERA_IP`
- Port: `5000`
- User: значение `ONVIF_USERNAME`
- Password: значение `ONVIF_PASSWORD`

Обычно вручную вписывать URL исходящего аудио не нужно: Onvifer должен увидеть RTSP Backchannel автоматически.

Если приложение всё же просит RTSP вручную, основной поток:

```text
rtsp://CAMERA_IP:8554/mainstream
```

Двухсторонний звук реализован через RTSP backchannel, а не через отдельный HTTP URL.

## Настройки по умолчанию v1.2.2

```sh
TIMEZONE="Europe/Kyiv"
LED_INDICATOR="off"
FULL_COLOR="off"
NIGHT_VISION="auto"
WDR="on"
WATERMARK="on"
VIDEO_FLIP="off"
VIDEO_FPS="25"

MEDIA_ENABLE="on"
BACKCHANNEL_ENABLE="on"
PTZ_ENABLE="off"

AUDIO_START_DELAY="120"
BACKCHANNEL_START_DELAY="20"
BACKCHANNEL_RESTART_DELAY="10"

WEB_AUTH="on"
FTP_ENABLE="on"
FTP_PORT="2121"
```

Для `Europe/Kyiv` есть совместимость со старыми системными zoneinfo: если на камере нет файла `Europe/Kyiv`, используется `Europe/Kiev`.

## Безопасность

- Перед публикацией или установкой не оставляйте `CHANGE_ME` в `config.sh`.
- FTP без TLS, используйте только в доверенной локальной сети.
- Веб-интерфейс использует логин и пароль из ONVIF-настроек.
- Не открывайте Web/FTP/ONVIF напрямую в интернет.

## Известные особенности

- После перезагрузки медиа-сервисы стартуют не сразу. Это сделано специально для стабильного звука и backchannel.
- Процесс backchannel может быть запущен постоянно, но динамик камеры играет только когда клиент реально отправляет аудио.
- Некоторые ONVIF-клиенты могут странно подписывать профиль кодека. Проверяйте фактический RTSP-поток.
- Это неофициальная прошивка/hack. Используйте на свой риск.
